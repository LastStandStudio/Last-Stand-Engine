Detailed Description

This is the mother class of the game. This Class represnts the entire game, thus its name "The Universe". Everything that goes on in the game will go on in the universe (this class). This makes levels and code eaier to manage.

Member Enumeration Documentation

enum Universe::EngineState

The diffrent modes and states that the Game Engine could possibly be in.

Enumerator
STARTING 

This is the very first state when the engine gets executed, used mainly for set-up. The atcual window creation is done in the Initialization step.

INITIALIZING 

At this state the engine is loading the window, all the calls in this state are low-level OpenGL calls to get everything running with the proper settings and prefrences configured.

RUNNING 

This is the Engine State that it should be in most of the time. In this state the programmer has the most amount of controll over the game and what it doing.

STOPING 

At this state the engine has been told to close one way or another. This is when everything should get to a stopping point because it will soon be removed from memory.
